Featured project

Dual Layer E Commerce Data Platform

This project started from a practical analytics problem. E commerce teams often need two things at the same time: real time visibility into activity and dependable historical data for reporting and finance. Many systems handle one well but not the other. I designed this platform to support both without sacrificing data quality.

Read the full breakdown ›

The platform has two coordinated layers. The first layer processes high frequency transaction events in near real time. I built a time window based processing flow that computes live business metrics such as sales velocity and inventory movement. Special care was taken to handle late arriving and out of order events so the metrics remain accurate instead of drifting over time. The system also triggers automated alerts when inventory thresholds are crossed so teams can respond quickly.

The second layer focuses on reliability and reporting. I built a batch ingestion and transformation workflow that aggregates daily transaction data into a structured warehouse model. Transformations are defined through versioned SQL models, and the pipeline is designed to be idempotent so retries do not create duplication. I added schema and business rule checks so invalid records are caught early instead of silently passing through.

The result is a system that supports both fast operational signals and stable analytical reporting. Dashboards update quickly, and downstream reports remain consistent and auditable.

Technologies used: Python, SQL, PostgreSQL, Kafka, dbt, Docker
