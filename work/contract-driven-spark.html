Contract Driven Data Platform with Spark

This project is about trust. If multiple sources feed the same model, someone needs to decide what “valid” looks like. I used contracts and schema validation so bad data gets caught early, not after it’s already polluted reporting tables.
Contracts + validation before analytics

What I built

  • Staged layers with predictable transformations
  • Schema + business rule validation on ingest
  • Incremental logic to avoid full reloads
  • Design focused on maintainability over time

Why it matters

When reporting is wrong, teams stop trusting the data and start rebuilding spreadsheets. My goal here was to make it hard for incorrect data to sneak in, and easy to explain what changed when it does.
