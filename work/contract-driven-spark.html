Project

Contract Driven Data Platform with Spark

A schema-first batch platform where data contracts and staged layers make multi-source ingestion safer and easier to maintain.

The problem

When multiple sources feed the same analytics tables, failures often show up late: a column changes shape, a field goes missing, and downstream dashboards quietly break. I wanted a system that catches issues at the boundary, not at the end.

What I built

A staged pipeline with explicit contracts. Data lands in a raw zone, gets validated against schema rules, then moves into curated layers. Transformations are predictable, and quality checks are part of the pipeline, not an afterthought.

Why the approach matters

Contracts reduce surprises: the pipeline knows what “valid” looks like and fails early when it doesn’t match.

Layering keeps things maintainable: raw, cleaned, and modeled layers make debugging and reuse much easier.

Incremental processing: avoids full reloads and scales better as data grows.

Tech stack

Spark, Python, SQL (plus validation checks and staged modeling)

Next improvements

Add automated contract versioning and richer observability so schema drift is visible before it becomes a production incident.
