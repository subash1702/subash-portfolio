Contract Driven Data Platform with Spark
Role Data Engineering
Stack Spark · SQL · Data Contracts · Layered Modeling
Focus Predictable transforms + validation

Contracts first. Then scale.

This project was about making multi-source batch data safer to use for analytics. I designed a layered pipeline where incoming data is validated against contracts before it ever lands in analytical tables.

What I built

  • Staged layers that separate raw ingestion, standardized staging, and final analytics models.
  • Contract checks to catch schema drift and breaking changes early.
  • Incremental processing to avoid unnecessary full refreshes as data volume grows.
  • A structure that makes transformations predictable and easier to maintain over time.

What I cared about

I wanted a pipeline where you can answer “why did this number change?” without panic. If the data changes, the system should either explain it—or block it.
