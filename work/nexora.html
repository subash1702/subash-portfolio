Nexora-NX — AI Knowledge Graph Builder

Nexora-NX converts unstructured text and PDFs into a structured knowledge graph by extracting entity–relationship triplets, building a NetworkX graph, and enabling explainable “how is X connected to Y?” queries backed by path evidence.

In simple terms: it turns messy documents into a navigable intelligence map.

The problem

Most knowledge in real teams lives in documents — PDFs, reports, notes, and raw text. They’re readable by humans, but not directly usable for structured analysis. The moment you want relationship discovery or “what connects X to Y,” it becomes manual scanning.

Nexora-NX turns unstructured documents into structured entity–relationship networks that can be explored, ranked, and queried.

What the system does

  • Ingests raw text or PDFs
  • Extracts entities and relationships via LLM triplets
  • Validates output into a structured schema (not freeform)
  • Builds a directed graph model with metadata
  • Visualizes the graph interactively in a Streamlit UI

What you can do with it

  • See relationship networks between entities
  • Rank important nodes using PageRank-style centrality
  • Find shortest paths between entities
  • Generate explainable answers using graph path evidence

Reliability choices I made

The hardest part wasn’t graph visualization — it was getting structured extraction that holds up across different inputs. I treated the LLM output like a data source: schema-first, validated, and retried when needed so the graph stays consistent.

What I’d improve next

Add entity resolution (dedupe near-duplicates), persist graphs so they survive restarts, and introduce evaluation metrics to measure extraction quality across domains.
