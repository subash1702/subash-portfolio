NYC Traffic Volume Analytics Pipeline

I built a pipeline to clean and align noisy urban traffic data into analytical tables that can answer practical questions: where congestion is growing, what patterns repeat, and what changes when major events hit the city.

Python
SQL
Geospatial
ETL
Modeling

The messy reality

Raw traffic datasets are rarely “analysis-ready.” Sensors drop readings, timestamps don’t line up, locations come in mixed formats, and the same street segment can appear with slightly different labels.

My focus was to make the data trustworthy first — then build analysis that doesn’t collapse the moment you zoom into a borough or a date range.

What I did

  • Standardized timestamps and removed duplicates / impossible values
  • Normalized location fields so segments match reliably
  • Built clean analytical tables (facts + dimensions) for repeatable queries

Why it matters

  • Teams can compare “apples to apples” across weeks and neighborhoods
  • Dashboards stay stable as new data arrives
  • Modeling becomes meaningful because the inputs are consistent

What I’d add next

A simple data quality layer that watches for missing sensor streaks and unusual distribution shifts, and a versioned schema so downstream users always know what changed.
