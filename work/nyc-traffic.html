NYC Traffic Volume Analytics Pipeline

A pipeline focused on turning messy urban traffic and event data into clean, consistent analytical tables. The main work was cleaning, aligning, and structuring data so analysis and modeling stop fighting the dataset.

Python SQL EDA Geospatial Model ready tables

Problem

Traffic data looks simple until you use it: missing sensors, inconsistent timestamps, and event signals that don’t line up cleanly with volume changes. The goal was to build a workflow that makes the data usable and explainable.

What I did

  • Cleaned and standardized timestamps and location identifiers
  • Created consistent aggregated tables for daily and hourly analysis
  • Aligned event signals with traffic volume trends for modeling and interpretation
  • Designed outputs so you can reuse them for dashboards or ML without re cleaning every time

Why it matters

This is the difference between “a dataset you explored once” and “a dataset a team can use every week”. I care a lot about that difference.
