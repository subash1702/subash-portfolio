Project

NYC Traffic Volume Analytics Pipeline

This project is about discipline with messy public data. The goal wasn’t just analysis — it was building a clean, reusable pipeline so the same dataset can support reporting and modeling without constant re-cleaning.

What I built

I standardized timestamps and keys, aligned traffic and event datasets, handled missingness and inconsistent formats, and produced stable analytical tables and feature sets that can be reused across models and dashboards.

How I think about this kind of work

The most expensive part of analytics is the part nobody sees — making data joinable, interpretable, and consistent. This pipeline focuses on predictable outputs and a structure that’s easy to maintain.
