NYC Traffic Volume Analytics Pipeline

Turning messy, multi-source city data into clean analytical tables and model-ready features.
Overview

NYC traffic datasets look simple until you try to use them: inconsistent timestamps, missing segments, event noise, and joins that explode row counts. This project focuses on building a pipeline that cleans, aligns, and structures raw traffic + event signals into tables that an analyst or model can trust.

What I built
  • A repeatable ingestion and cleaning flow with consistent time keys and location identifiers.
  • Feature engineering for daily/weekly seasonality and event-impact windows.
  • Analytical tables designed for both dashboards and predictive modeling.
  • Clear documentation so a reviewer can follow the logic end to end.
How I think about it

Instead of optimizing for “a nice chart,” I optimized for the table underneath the chart. If the base table is stable, everything downstream becomes easier: BI, modeling, and debugging.

Stack
  • Python for cleaning + transformations
  • SQL for structured joins and aggregations
  • Visualization/EDA to validate assumptions and data quality
