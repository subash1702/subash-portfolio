Project

NYC Traffic Volume Analytics Pipeline

A pipeline that turns noisy, inconsistent public traffic data into clean analytical tables and reusable features for modeling.

The problem

Public datasets are useful, but they’re rarely “analysis ready.” Formats differ, timestamps don’t align, and missing values create false patterns. The goal here was to build a clean foundation so analysis and modeling don’t become a constant cleanup exercise.

What I did

I cleaned and standardized traffic and event datasets, aligned them on time and location, and produced consistent tables for reporting and feature sets for predictive models. The output is designed to be reused without redoing the messy prep work.

How I made it reliable

Standardized keys and timestamps: so joins behave predictably and time-based aggregations are correct.

Clear transformations: the pipeline is readable and debuggable instead of being a one-off notebook.

Reusable outputs: clean analytical tables and features that can power dashboards and models.

Next improvements

Add automated validation for new incoming files and publish a small data dictionary so the tables are self-explanatory.
