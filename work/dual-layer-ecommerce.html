Project

Dual Layer E-Commerce Data Platform

I built this because teams usually need two things at once: fast operational signals and stable, auditable reporting. This project is my way of showing how I design for both without cutting corners on correctness.

Architecture in one paragraph

Events stream through Kafka. A Python processor computes windowed metrics and stays correct when events arrive late. Daily logs land in PostgreSQL, and dbt transformations build reporting tables that remain consistent across retries. Data quality checks validate schema and business rules before anything hits BI.

What I cared about

Not just “making it work”—making it safe to operate. Late events, duplicates, and retries are normal in real systems, so I designed with those realities in mind from day one.

Tech

Python, SQL, Kafka, PostgreSQL, dbt, Docker
