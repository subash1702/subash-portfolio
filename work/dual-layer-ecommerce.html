How I work

I start by asking what decision the data needs to support, then I work backwards: what should be true about the data for someone to trust it without second-guessing.

I like systems that stay correct as they scale — idempotent pipelines, clear contracts, and checks that fail loudly when something breaks upstream.

When I’m building, I keep things simple: one clean source of truth, predictable transformations, and documentation that a new teammate can follow.
